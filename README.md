# 🚗 Processing 3D Traffic Simulation

## Overview

This project is a real-time 3D traffic simulation built in **Processing (Java mode)**.

The goal of the project is to explore game architecture, behavioral systems, and real-time simulation logic without relying on a full game engine like Unity or Unreal.

The simulation focuses on:

- Vehicle AI behavior phases  
- Lane switching and overtaking logic  
- Speed balancing and adaptive reactions  
- Distance tracking and score calculation  
- Real-time rendering using P3D  
- Custom UI layered on top of a 3D world  

This project is part of my ongoing journey from long-time hobby programmer to formal studies in **System Development / Secure Software Development**.

---

## 🧠 Technical Focus

The project explores:

- Object-oriented design in Java  
- State-driven vehicle behavior  
- Real-time update loops (`draw()` lifecycle)  
- Array-based entity management  
- Separation of logic and rendering  
- Performance considerations in GPU-accelerated P3D   

---

## 🎮 Why Processing?

Instead of using a traditional game engine, this project intentionally uses Processing to:

- Understand rendering pipelines at a lower level  
- Build update loops manually  
- Control entity lifecycle explicitly   

It serves as both a learning tool and a technical playground.

---

## 🛠 Features

- Multi-vehicle simulation  
- Overtake and merge behavior  
- Dynamic speed adaptation  
- Player-relative movement system  
- Custom scoring logic (distance + speed + time)
